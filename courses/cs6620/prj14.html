Teapot Rendering Competition Submission

For the final neat feature I chose to implement participating media with single scattering in the form of smoke and steam. This allows for creating nice atmospheric effects and a steaming teapot. I also have support for physically based materials in the form of microfacet models, measured BRDF data from the MERL BRDF Database and measured metal properties from PBRT's SPD files, although this was implemented a bit back. Material properties can also be chosen based on a texture allowing for creation of the tile floor, which is made of rough copper and white diffuse tiles. As an extra bonus the renderer is also lock-free, so performance is ok. The image was rendered using path tracing with adaptive sampling using 128 up to 2048 samples and took TIME HERE time to render.

Hardware Used and Other Details

Render times were measured using std::chrono::high_resolution_clock and only include time to render, ie. time to load the scene and write the images to disk is ignored.

CPU: Intel i7-4790k @ 4.0Ghz
RAM: 16GB 1600Mhz DDR3
Compiler: gcc 4.9.2 x86_64 (on Windows, built by MinGW-W64 project)
Compilation Flags: -m64 -O3 -march=native -flto -fno-exceptions -DNDEBUG
